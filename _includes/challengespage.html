Data Challenges by Sponsors

Here are some of the challenges you can choose to tackle during the event:




Query description – One of the ways to navigate through ebay’s inventory is by browsing the category tree and then restricting it by certain attributes (from the kind I described above). Each combination of category and aspects is actually a query (for example category =digital cameras, brand = sony, type =dslr, with lens = yes ). We would like to produce a natural language description for this query (“Sony DSLR Cameras with Lens”). We will provide one version in English and one version in German.

Category recommendation – When a seller lists something on ebay, he has to provide a category. eBay assists him by suggesting a category based on the short title the seller provides. However, due to the very large number of categories and some ambiguity we have in the category tree, this suggestion can be difficult. In this challenge, we will provide a big dataset of item titles with they’re corresponding categories. The challenge will be to classify a test set of titles to they’re correct category.

Matching quality score – For managing its vast inventory, eBay has created a large platform of catalogs and classifies it’s listings to cataloged products. In this challenge we ask you to predict whether the matching of a listing to a product is correct. We will provide a medium sized sample of pairs of matchings between a listing and a product, with various information about both the item and the product – things like free text titles, item’s domain, seller information and some of our internal algorithms outputs. The goal here is to predict whether the item matches the product or not.

Item clustering - eBay is very flexible on the data it requires the sellers to provide. Upon listing, the seller provides a short title of what he sells, some optional domain specific specifications (like screen size for TV’s, color for fashion or clarity for diamonds), and a category, an image and an HTML formed description. With that data we would like to cluster the items to groups which represent the same real world products.

Image selection – Providing a good image is a key factor in selling on eBay. A good image should be clear, without watermarks, with the object in the center, hopefully with white background and so on. In this challenge we will provide groups of items with they’re corresponding images. For each group we would like to get the best image. If the participants want to modify/correct/synthesize an image, this can be even cooler. :)

NV extraction – Metadata enables a variety of backend processes as well as improves the user experience on eBay. However, due to the huge variety of eBay’s inventory, both collecting and extracting the metadata is a big challenge. We will provide a dataset from a small subset of domains, where for each domain we provide a set of item titles and a dictionary of metadata values. Here we ask to extract the correct value from the title and potentially to enrich the dictionary. As an example we may provide a dataset of TV’s and a dictionary of brands and models, where the goal will be to tag each listing with a brand and model from the dictionary or potentially identify new brands that are absent.







Given two users and two articles, can you tell what user read which article? Using obfuscated data extracted from Outbrain databases, participants are expected to build models and combine them to achieve the best predictor. The data contains selected features of all relevant articles, and a set of article viewing events by various users including page referrals.







The Wix platform allows users to build their own websites and publish them. We provide the text of those sites in a format close to the original one. For each site we give you a categorization of that site. Using the text content data, build a model using ML / NLP that will categorize the sites. This model will be expected to work on unseen data. The best result for the unseen set wins!
